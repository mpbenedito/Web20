DOM - Documento Object Model

É uma API multiplataforma acessível pelo javascript pelo qual consigo referenciar os elementos que estão na página.

Uma página html é o resultado da interação de diversos objetos que são acrescentados e parametrizados por meio de tags que encapsulam conteúdos, que são outras tags ou elementos de texto.

Quando o navegador baixa o conteúdo de uma página ele está fazendo o download de um texto. Logo o navegador precisa organizar todo este conteúdo para apresentar uma saída formatada.

Todos os elementos HTML quando interpretados pelo navegador ficam armazenados no DOM, cuja representação lembra uma árvore (Árvore de Elementos da Página).

Estes elementos também são chamados de nós ou "nodes". Um exemplo é a tag <html>. Ela encapsula as tags <head> e <body>. Então é dito que estas são filhas de <html> e irmás entre si.

Falar do DOM é falar desta árvore de elementos. A API de acesso ao DOM fornece métodos que propiciam a alteração dos atributos dos elementos.

Selecionando elementos de uma página

Uma página carrega nativamente o elemento document. E é este objeto que possui métodos específicos para acesso a todos os elementos carregados na página.

Principais métodos: getelementbyid(), getelementsbytagname(), getelementsbyclassname(), getelementsbyname()

O input abaixo possui os seguintes atributos: type="text", placeholder="Digite seu nome", id="nome_usuario", class="campo_texto" e name="nome"



Importante: o navegador utiliza um método de interpretação para reconhecer os elementos. Isto significa que o head da página sempre é interpretado e reconhecido antes do body. Se o script que referencia o DOM for colocado no head ou antes que o elemento referenciado tenha sido criado ocorrerá um erro.

O getElementById() vai selecionar o input buscando pelo Id "nome_usuario". O Id sempre deve ser valor único na página. Ponto de atenção: o método getElementById() é no singular mesmo enquanto os outros métodos são no plural (getElementsBy...). Isto faz com que o getElementById() busque sempre um HTML Element. Os outros métodos sempre buscarão HTML Collections.

Os métodos getElementsby... é preciso tomar cuidado pois não será retornado um único elemento, mas vários que se enquadrem na mesma característica procurada.

O getElementsByTagName() deve ser utilizado procurando os elementos pelo nome de sua tag. No exemplo, pela tag "input".

O getElementsByClassName() deve ser utilizado procurando os elementos pelo nome de sua tag. No exemplo, pela class "campo_texto".

O getElementsByName() deve ser utilizado procurando os elementos pelo seu atributo "name". No exemplo, pela class "nome".

Importante: O método getElementsByName() recupera os elementos da página colocando-os numa nodelist.

  • getElementById() - retorno: HTML Element

  • getElementSByTagName() - retorno: HTML Collection

  • getElementSByClassName() - retorno: HTML Collection

  • getElementSByName() - retorno: HTML Node List

Esta diferenciação é importante para os tratamentos que devem ser dados em cada situação.
