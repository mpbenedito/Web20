Introdução

ECMAScript (ES) é uma especificação de implementação da linguagem para padronização entre os vários navegadores e suas versões, pois no começo da linguagem, cada fabricante de browser desenvolveu sua própria "interpretação" da linguagem. Rapidamente a linguagem javascript deixaria de ser útil, pois se cada navegador desenvolvesse suas particularidades na interpretação dos códigos, não seria possível adotá-la como linguagem padrão para desenvolvimento.

Em 1997 o javascript foi entregue para o ECMA (European Computer Manufactoring Association), um órgão terceiro e independente, para que uma padronização pudesse ser implementada e divulgada, á qual os navegadores deveriam processar e interpretar, de forma análoga.

Após a padronização das sintaxes e semântica o js ganhou a robustez de uma linguagem mais industrial, ou seja, de uso generalizado. Isso aumentou a compatibilidade de aplicações rodando em browsers diferentes.

ES6 x ES5

Embora a versão mais atual do ECMAScript seja a versão 6 (ES6) os fabricantes de browsers não implementaram totalmente suas funcionalidades, semânticas e sintaxes. Com isso surgiram no mercado vários interpretadores de código (transpirers) que convertem um fonte coidificado em ES6 e faz um parser, transformando a codificação para o padrão ES5.

Todos estes estudos foram desenvolvidos e testados utilizando o ES6 porém com o compilador BABELjs, que é um compilador de mercado e é bastante utilizado.

Quando os navegadores rodarem nativamente o ES6, provavelmente a ES7 já terá sido homologada, quando o babel deverá ser atualizado.
