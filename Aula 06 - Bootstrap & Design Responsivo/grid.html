Bootstrap4 - Sistema de Grid

O Bootstrap 4 tem por padrão uma divisão interna que possibilita a divisão de um site em até 12 partes.

Na definição do projeto monta-se o site imaginando quantos espaços cada seção do site irá ocupar deste grid

Feita esta elaboração aí inicia-se a divisão dos espaços montando como se fosse um croqui de leiaute, que irá ajudar a definir a formatação de tudo.

Exemplo de leiaute aplicado

Esta divisão define o número máximo de grids que cada seção irá ocupar. As classes do bootstrap estão preparadas para esta definição, desde que o total seja sempre 12.

Os grids não aparecem, são epanas para organização dos espaçamentos.

Estas áreas não tem largura fixa. Elas sempre existirão em qualquer tamanho de dispositivo, e cada segmento do grid terá sua área ajusatada para cada tamanho de tela.

O site do Bootstrap utiliza este sistema de grids

Aplicando grids

1) Definição do container: classe "container". A classe container, por si, já será centralizada. Se o tamanho da tela for reduzido o container também reduz.

2) Definir verticalmente quantos espaços que o site irá ocupar. Classe row, que irá ocupar todo o espaço do container que ela está classificando

3) Fazer a divisão dos conteúdos informando quantos espaços cada div irá ocupar. Classe col-[tamanho]-[n-espaços], onde:

a) [tamanho] = (não tem)/sm/md/lg/xl para definir o tamanho do monitor mínimo para exibição (ver media queries e breakpoints)

b) [n-espaços] = quantidade de grids que a divisão irá ocupar.

Offset

Por padrão o row sempre ocupará todo a área útil do container.

Porém, existem situações que é necessário que as rows ocupem espaços com deslocamento no grid.

Nestes casos utiliza-se a classe offset-[tamanho]-[número], onde número é a quantidade de espaços do grid que devem ser deixados antes do primeiro elemento da row ser colocado.

No exemplo abaixo, é acrescentado um offset de 3 espaços para telas maiores que o 760px. Para telas menores a row ocupará os 12 espaços do grid

Aninhamento

É possível realizar a divisão de uma área que seja de uma row, que utilize tantos espaços de grid quanto se deseja, em um novo grid de 12 espaços

Esta é a principal utilização do conceito de que um uma row sempre terá o espaço de seu container, não importa o tamanho deste container. E este espaço todo sempre será um novo grid de 12 divisões iguais.

Na prática, é colocar row dentro de row, dentro de row e definir, pelos breakpoints quando deve ocorrer o empilhamento.
