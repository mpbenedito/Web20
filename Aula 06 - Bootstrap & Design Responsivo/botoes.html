Botões

Inicia-se com a classe padrão: btn

Botões simples com background coloridos: btn-primary, btn-secondary, btn-success, btn-info, btn-warning, btn-danger, btn-light, btn-dark, btn-link

Botões com contorno

Classes: btn + btn-ouline-[formato], onde [formato] = primary / secondary / success / info / warning / danger / light / dark / link

Tamanhos dos botões

Classes: btn + btn-[tamanho], onde [tamanho] = lg (large) / sm (small) / block (bloco]



Estados de botões

Indicar se os botões estão desabilitados ou ativados:

Classes: btn + btn-[cor] + [estado], onde [estado] = active / disabled

Grupos de botões - horizontal

Cria uma barra horizontal de botões. Colocar os botões num container e aplicar a classe btn-group. Os botões podem ser formatados individualmente

Grupos de botões - vertical

Cria uma barra vertical de botões. Colocar os botões num container e aplicar a classe btn-group-vertical

Toolbar de botões

Criar arranjos diferentes de botões com o agrupamento horizontal (btn-group) aplicado em containeres distintos.

Estes arranjos ainda podem ser aproximados se tiverem um outro container externo com a classe btn-toolbar

Botões dropdown

Botões que tornam-se visíveis ao pressionar outro botão, como um menu combo.

Classes: dropdown, deve ser atribuída a classe dropdown no container de todo os elementos

O botão que esconderá os elementos deve ter a classe dropdown-toggle e o atributo data-toggle definido como dropdown

O container que manterá os elementos escondidos deve ser classificada como dropdown-menu.

Por último, os elementos escondidos no botão devem ter a classe dropdown-item atribuída

Um container com classe dropdown-divider cria uma linha de separação entre os dropdown-item.
