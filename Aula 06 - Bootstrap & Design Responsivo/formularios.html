Formulários Estilizados com Bootstrap

Elementos com características de display block.

Perceber que existem vários tipos de elemento de formulários e cada um receberá formatações específicas.

Perceber que os elementos se adaptam ao tamanho da tela.

form-group: aplicada ao container irá isolar os elementos do container em um espaçamento pré-formatado.


form-control: aplicado aos inputs apresentará o comportamento dos elementos, estilizando-os


form-control-[tamanho]: Onde o tamanho são duas opções sm (small/pequeno) ou lg (large/grande)


Perceber que o email está desabilitado, porque ele está com atributo readonly.

A observação "Email será validado" está em tamanho menor, pois está estilizado com a tag small

Email será validado

Caixa de seleção: aplicar o form-control no elemento select


Textarea: aplicando of form-control ao elemento textarea


A classe para inputs do tipo file é o form-control-file, que irá estilizar o elemento da forma correta.

A label após o elemento que apresenta o arquivo selecionada é automática do elemento.


Existe uma segunda formatação para o input de arquivos que pode ser adotada também pelo bootstrap.

Classes: custom-file, custom-file-input e custom-file-label




Formatação de botões do tipo range (slide)


Elementos com características de display inline.

Classe aplicada diretemente no formulário: form-inline



Formatações simples de elementos de fomulários

Ainda não é validação. São apenas formatações para aplicar aos elementos após eles serem validados

Classes para os inputs:

is-vald - indca que o elemento passou pela validação com sucesso adicionando uma borda verde no elemento e o sinal de chek

is-invalid - indica que a validação do elemento não foi bem sucedida adicionando uma borda vermelha no elemento e o x vermelho

Para as mensagens de válido ou inválido é possível utilizar as classes valid-feedback e invalid-feedback

Senha inválida (invalid-feedback)
